常用的 Linux 指令

1. 你常用的 Linux 指令有哪些？請說出檔案、程序、網路、系統資源各 3～5 個。

平常在處理 檔案 的話，最基本的就是 ls 看一下目錄、cd 切換路徑，要搬東西或改名就用 mv，刪除或建立目錄就用 rm 跟 mkdir。如果想直接看檔案內容，我通常會用 cat 或 less，找東西的話 grep 跟 find 絕對是標配。

程序管理 的話，我習慣用 htop（介面比較好讀）或 top 看一下誰在吃資源，要砍掉失控的程序就直接 kill。

網路 部分，最常用 ip a 看網卡資訊，ping 測試有沒有通，或是用 ss 跟 netstat 查一下 Port 有沒有被佔用。下載檔案或測試 API 就交給 curl 或 wget。

最後是 系統資源，我會習慣用 df -h 看硬碟剩多少，free -h 看記憶體夠不夠用，還有 uptime 瞄一下系統載重（Load Average）。

系統負載等資源查看

2. 如何查看 Linux 系統負載、記憶體、磁碟使用率？
  • 系統負載：uptime, top, htop
  • 記憶體：free -h, vmstat
  • 磁碟使用率：df -h
  • 資料夾大小：du -sh

查看系統 Log

3. 如何查看系統 Log？
  • 系統日誌：journalctl -u <服務名>
  • 開機/硬體：dmesg
  • 應用程式：通常在 /var/log/ 下，依服務不同。

管理系統服務

4. 如何管理系統服務？
  • 管理操作：systemctl start/stop/restart/status/enable <服務名>
  • 啟用開機自動：systemctl enable <服務名>

網路相關指令

5. 網路相關指令
  • ip：查看/設定網路
  • ping：測試連通性
  • ss / netstat：查看連線、Port
  • traceroute：查看路由節點

Port 佔用查詢

6. 如何查詢 port 是否被佔用？
  • ss -tulnp | grep <port>
  • netstat -tulnp | grep <port>

檔案權限理解

7. 檔案權限：chmod、chown、rwx 怎麼理解？
  • 權限數字：r=4, w=2, x=1
  • chmod 755 <檔案>：設定擁有者rwx，群組及其他rx。
  • chown user:group <檔案>：更改檔案擁有者與群組。

服務起不來排查

8. 如何排查 Linux 上服務起不來？

如果 服務起不來，我第一步一定是先下 systemctl status 看它噴什麼錯。如果資訊不夠，就去翻 journalctl -u 看詳細日誌。很多時候其實只是設定檔寫錯，所以我會先檢查語法，或是看 Port 是不是被別人搶走了。

Linux 開機程序

9. 請簡述 Linux 的開機程序？

Linux 的開機程序大致可以分為以下階段：

  1. BIOS / UEFI (硬體自檢)：開機後進行 POST (Power-On Self-Test)，檢查硬體資源並尋找開機裝置。
  2. MBR / GPT：讀取開機裝置前段的開機管理程式。
  3. Boot Loader (如 GRUB2)：提供選單選擇作業系統，並將指定的核心 (Kernel) 與 Initramfs 載入記憶體。
  4. Kernel & initramfs (核心載入)：核心接管硬體，載入必要的驅動程式，並掛載真正的根目錄 (/)。
  5. init / systemd：核心呼叫系統的第一支程式 (PID=1)，載入對應的 target 初始化系統環境。
  6. 啟動服務與登入介面：根據設定啟動各項背景服務，最後提供 Login 提示字元或圖形介面。

磁碟分割規劃

3. 如何規劃磁碟分割？

一般會依用途進行磁碟分割规划，例如：

  • /：根目錄，系統核心與多數程式安裝處。
  • /home：使用者資料區，與系統隔離避免重灌遺失資料。

遠端管理與安全

4. 遠端管理：SSH 設定、金鑰驗證、安全性基本觀念。
  • 使用金鑰登入，取代純密碼。
  • 關閉 root 的預設密碼遠端登入 (PermitRootLogin no)。
  • 修改預設的 SSH Port (例如改掉預設的 22 port)。
  • 配合防火牆限制，只允許白名單 IP 連線 SSH。

常見問題排查

5. 伺服器常見問題：開機失敗、網路不通、服務掛掉怎麼排查。
  • 開機異常：看 BIOS 提示、dmesg、檢查硬碟/記憶體。
  • 網路不通：檢查 IP、閘道、DNS、防火牆。
  • 服務掛掉：看日誌(journalctl)、系統資源使用率(top, free)、檢查設定檔。

網路基礎名詞解釋

1.請解釋：IP、子網路遮罩、閘道、DNS。
  • IP：網路上主機的位置/位址。
  • 子網路遮罩：用來區分內部網段與外部網段。
  • 閘道：當封包要前往其他網段時，對外的預設出口。
  • DNS：負責將域名轉化為 IP 位址的服務。

網路不通排查流程

2.客戶說「網路不通」，你標準排查流程是什麼？

碰到 網路不通，我有一套標準流程：

先看自己有沒有抓到 IP。

ping 閘道（Gateway），確認內網是通的。

ping 8.8.8.8 看能不能出外網。

如果 IP 能通但網址不通，那八成就是 DNS 掉漆了，我會去檢查 /etc/resolv.conf。

ping 不通的原因

3.ping 不通有哪些可能？
  • IP / 子網路遮罩 / 閘道 等設定錯誤。
  • 被沿路的防護牆阻擋 (如阻擋了 ICMP 封包)。
  • 對層設備主動關閉了 ICMP 回應。
  • 實體網線沒接好或網卡本身異常。

DNS 異常判斷

4.DNS 異常怎麼判斷？
  • 如果 IP 可以 ping 通，但 ping 域名卻找不到主機，那多半就是 DNS 的問題。
  • 可以檢查 /etc/resolv.conf 裡的 DNS server，或者系統的網卡 DNS 設定是否正確。

防火牆基本觀念

5.防火牆基本觀念（firewalld /ufw/iptables）

至於 防火牆，現在主流不是 ufw 就是 firewalld，我的原則就是「白名單模式」，除了必要的 Port 像 80 或 443 以外，剩下的通通鎖起來，這樣才安全。

網路與路由觀念

6.區域網路、對外網路、路由基本觀念。
  • 區域網 (LAN)：局域網路內部的電腦互通，不需透過外部路由。
  • 對外網 (WAN)：需要透過閘道器/路由器，將封包送往外部網際網路。
  • 路由 (Routing)：當網路封包要從 A 到 B 時，決定封包要走哪一條路徑送達的機制。

虛擬機與 Hypervisor

1. 什麼是虛擬機？什麼是 Hypervisor？
  • 虛擬機 (VM)：在實體電腦上透過軟體模擬出的獨立主機環境，擁有自己的 OS。
  • Hypervisor：在實體硬體和虛擬機之間的管理層，負責分配資源給虛擬機（例如 VMware, KVM, Hyper-V）。

用過的虛擬化技術

2. 你用過哪些虛擬化？（VMware / Hyper-V / KVM / VirtualBox）
  • 本地端常用 VMware Workstation、VirtualBox。
  • 有接觸過 KVM 虛擬化平台。
  • 雲端 VM 環境也熟悉，如 AWS EC2 等。

虛擬機網路模式

3. 虛擬機網路模式有哪幾種?
  • 橋接模式 (Bridged)：VM 視同實體機，與宿主機在同一個網段，能直接取外部 IP。
  • NAT 模式：VM 透過宿主機網路當作跳板分享上網，外部連不進 VM (需做 Port Forwarding)。
  • 僅主機模式 (Host-Only)：VM 只能與宿主機連線，無法直接上網。

VM 常見問題處理

4. VM 常見問題： 開不起來 網路不通 磁碟空間不足 你怎麼處理？
  • 開不起來：檢查映像檔是否有損毀、資源配置是否超出實體極限、Hypervisor 後台服務是否正常。
  • 網路不通：檢查網路模式(NAT/Bridged)、IP 是否正確、是否有防護牆阻擋。
  • 磁碟滿：先清理無用大檔案，若空間仍不夠，則透過 LVM 擴增磁碟容量。

設備連不上伺服器

1. 使用者電腦 / 設備連不上伺服器，你怎麼排查？
  1. 先確認實體的網路線或 WiFi 訊號正常連接。
  2. 檢查本機的網路設定是否正確獲取到 IP。
  3. 用 ping 測試伺服器 IP 看看兩端是否連通。
  4. 如果能 ping 通，則檢查後端服務的埠號狀態、防護牆規則是否擋住了，以及伺服器服務是否真正啟動運行中。

網卡與 WiFi 異常

2. 網卡異常、無法抓 IP、連不上 WiFi 怎麼處理？
  • 檢查網卡的驅動程式是否有遺失或錯誤驚嘆號。
  • 檢查網線以及交換機狀態是否亮燈正常運作。
  • 若抓不到 IP，檢查內部 DHCP 服務是否正常。
  • 可以嘗試將網卡停用後再重新啟用，或者釋放並重新獲取 IP。

簡單硬體故障判斷

3. 簡單硬體故障判斷：開機無畫面、風扇轉但不開機等。
  • 開機無畫面：如果是主機通電但無畫面，可先檢查電源供應是否足夠、顯示卡或螢幕線材有無接觸不良、以及記憶體重新插拔。
  • 風扇轉但不開機：主機板通常會有錯誤燈號提示（如 CPU、DRAM、VGA 燈）、或者發出 BIOS 逼逼聲，最常見的處置為將記憶體拔下擦拭金手指並重新安插。

服務起不來怎麼查

1.服務起不來怎麼查？
  1. 使用 systemctl status <服務> 來查看服務狀態與錯誤提示。
  2. 使用 journalctl -u <服務> 查詢詳細服務啟動日誌。
  3. 確認設定檔有無寫錯 (例如 nginx -t)。
  4. 確認監聽的 Port 是否被佔用，並檢查相關權限。

磁碟滿了怎麼清

3.磁碟滿了怎麼清？

磁碟爆掉 的話，我會先用 df -h 定位是哪個掛載點滿了，然後進去用 du -sh * 一層一層找哪個資料夾是吃空間的元兇。通常都是老舊的 Log 或是沒清掉的備份檔，確認沒用就直接清掉。

程序掛掉 / CPU 爆了

4.程序掛掉 / CPU 爆了怎麼處理？
  1. 輸入 top 或是 htop 找出這時哪個程序佔用了大量 CPU 或記憶體資源。
  2. 若是無主或失控的異常程序，下指令 kill <PID> 強制中止。
  3. 查看應用程式的專屬 Log 以及系統 dmesg 或 journalctl 日誌，分析掛掉的根本原因 (如設定錯誤、OOM 或硬體負載過載)。
