Q3 — 開發的系統規模多大？

C/C++ 言語特性 Q5〜9

Q5 — 使用C++哪個版本？

主に C++17 を使用していました。モダンな機能を活用することで、コードの安全性と記述の簡潔さを両立させています。

Q6 — 如何活用C++ Modern功能？

std::unique_ptr や std::shared_ptr などの スマートポインタ を活用し、手動の delete を避けることでメモリリークを徹底的に防ぎました。また、std::vector などの STL容器 を使い、メモリ管理の安全性を高めています。

Q7 — 如何防止Memory Leak？

スマートポインタの使用に加え、RAII原則（Resource Acquisition Is Initialization）を徹底しました。また、開発の最終段階では必ず Valgrind 等のツールでメモリチェックを行っていました。

Q8 — std::vector 和 std::array 怎麼選用？

要素数が動的に変わる場合は std::vector を使用し、要素数がコンパイル時に決まっている場合は std::array を使用します。車載システムのようなリソースが限られた環境では、スタックメモリを利用する std::array の方がオーバーヘッドが少なく、好ましいケースが多いと考えています。

Q9 — C和C++怎麼選用？C++的優點是什麼？

C言語はハードウェアに近い制御に向いていますが、C++は カプセル化 や テンプレート を用いることで、高いパフォーマンスを維持しつつ、コードの再利用性と保守性を向上させられる点が大きなメリットだと考えています。

低レベル最適化・並列化 Q10〜15

Q10 — 使用什麼Profiling工具？

CPUの処理に関しては gprof や perf を使用し、GPUに関しては NVIDIA の Nsight Systems や Nsight Compute を用いて、カーネルの実行時間やメモリ帯域の利用効率を分析していました。

Q11 — 最佳化Cache時注意什麼？

空間的局所性 を高めるために、データのレイアウトを「配列の構造体（AoS）」から「構造体の配列（SoA）」に変更し、連続したメモリ領域へのアクセスを増やすことでキャッシュミスを最小限に抑えました。

Q12 — OpenMP 和 pthreads 怎麼選？

ループ処理などの単純な並列化には、実装コストの低い OpenMP を使用します。一方で、スレッドの優先度設定や、より細かいライフサイクル管理が必要な複雑なタスク並列化には pthreads を使用しています。

Q13 — CUDA實作中如何減少資料傳輸Overhead？

ホストとデバイス間の転送を最小限にするため、なるべく多くの処理をGPU側で完結させるようにしました。また、cudaMemcpyAsync を使用して、計算と通信をオーバーラップさせる（隠蔽する）工夫も行いました。

Q14 — 如何活用Shared Memory？

グローバルメモリへのアクセス頻度を減らすため、ブロック内のスレッド間で頻繁に再利用されるデータを Shared Memory に一度ロードし、高速なアクセスを実現することで全体の処理時間を短縮しました。

Q15 — 如何避免Race Condition？

基本的には std::mutex や std::lock_guard を使用しましたが、パフォーマンスが極めて重要な箇所では、より軽量な std::atomic（原子操作）を使用してロックフリーな設計を試みました。

車載・組込みシステム Q16〜19

Q16 — 為什麼車載軟體要有「即時性」？

自動ブレーキやステアリング制御などの機能は、ミリ秒単位の遅延が人命に関わる事故に直結するためです。処理が「速い」だけでなく、常に「一定の時間内に終わること（決定論的な動作）」が保証される必要があります。

Q17 — Memory受限環境中如何維持性能？

動的なメモリ割り当て（malloc/new）を極力排除し、静的なメモリ確保 を行うことです。これにより、フラグメンテーションを防ぎ、実行時の予測可能性を高めることができます。

Q18 — 知道MISRA C/C++等規範嗎？

実務での使用経験はありませんが、未定義動作を避け、コードの安全性を高めるための厳格なルールであると認識しています。車載エンジニアとして、これらの規約に従った高品質なコードを書く準備はできています。

Q19 — 怎麼確保自己程式碼的品質？

Google Test を用いた 単体テスト を自動化し、カバレッジを意識した検証を行っていました。また、境界値テストを行い、予期せぬ入力に対しても堅牢なコードであることを確認していました。
